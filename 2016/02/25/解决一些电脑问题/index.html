ubuntu 终端中文乱码

ubuntu iterm 连接之后, 终端无法输入中文 , vim 里面的中文都是乱码, ubuntu tty 显示中文为乱码

解决方法:

添加中文字符编码：
$sudo vim /var/lib/locales/supported.d/local
#添加下面的中文字符集, 一开始是只有en_US.UTF-8 UTF-8这一行的
zh_CN.GBK GBK
zh_CN.GB2312 GB2312
zh_CN.GB18030 GB18030
使其生效：
$sudo dpkg-reconfigure locales
然后重启, 或者重新登录终端就 ok 了

Error checking TLS connection: Host is not running

mac 上用 Dockertool 安装了 Docker, 我把eval $(docker-machine env default) 加到了 .bashrc 里面, 但是当网络发生变化的时候, Docker 的 ip 会出现问题 ,连接不上, 于是会提示这个错误

把上面那句改成下面的就解决了:

# connect your shell to docker-machine
[ Running == `docker-machine ls | awk 'NR==2{print $4}'`  ] && eval $(docker-machine env default) || echo fuck

mac os 还原最小化窗口的快捷键

最小化快捷键 command + m

发现用cmd + tab键不能还原最小化的窗口，非常不方便，Google了一下找到一个最简单的方法。

用cmd+Tab键选中要还原的程序后，保持cmd键不放，松开tab键再按住alt/option 键，再松开cmd键最后松开alt/option键。

试试吧。

/var/log/messages 日志文件里大量的: systemd: Started Session 1858 of user root.

解决方法:

vi /etc/systemd/system.conf
LogLevel=notice

然后

systemctl restart rsyslog
systemd-analyze set-log-level notice
