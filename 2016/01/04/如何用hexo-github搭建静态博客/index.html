如何用hexo-github搭建静态博客

  • github
  • hexo
  • os x
it
文章目录
  1. 1. 一 安装nodes
  2. 2. 二 安装初始化配置 hexo
  3. 3. 三 配置 github page
  4. 4. 四 部署到github
  5. 5. 五 写文章
    1. 5.1. 草稿
    2. 5.2. 预览

搭建环境和版本:

  • OS X EI Captain

  • node v4.2.4

一 安装nodes

brew安装NVM

brew install nvm

將下列指令加入 .bash_profile（或 .bashrc）檔案

重新載入 .bash_profile 設定

source .bash_profile

找出目前所有可安裝的 Node.js 版本

nvm ls-remote

安裝最新稳定版的 Node.js(我这时候是v4.2.4)

nvm install v4.2.4

指定nvm使用的 Node.js版本

nvm use 4.2.4

預設使用 0.11.16 版本，否則每次重新連線登入，還需要重新 nvm use 一次

nvm alias default 4.2.4

測試 Node.js

node -v

二 安装初始化配置 hexo

初始化

$ hexo init [folder]      #创建 hexo 主目录

生成静态网页文件

$ hexo g

本地运行服务查看,访问 http://localhost:4000

$ hexo s

三 配置 github page

注册 github 账号
新建一个仓库, 仓库名: yourusername.github.io (必须是这个格式)

四 部署到github

打开 _config.yml 文件, 修改

然后部署

$ hexo clean   #清空缓存
$ hexo d

如果提示

ERROR Deployer not found: git.

安装

npm install hexo-deployer-git --save

即可解决

部署完成后, 访问 http://yourusername.github.io 就可以看到效果了

五 写文章

$ hexo new "文章标题"

然后在 source/_posts/ 目录下可以找到文件进行编辑
编辑完成后,部署更新:

$ hexo g -d

基本这样就能写了

草稿

hexo n 默认是创建正式发布的文章, 想要先写草稿不发布, 使用这样的命令创建草稿:

hexo n draft <title>

预览

预览正式文章

hexo s

预览包含草稿在内的文章

hexo s --draft

将草稿正式发布

hexo publish <title>

注:草稿 publish 之后, 如果要变更内容, 记得要在 post 的位置重新打开文章, 因为文章位置变了, 继续原来的编辑会导致保存不了
