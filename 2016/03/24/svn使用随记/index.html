创建仓库

svnadmin create /data/svndata/winne  # winne为版本库名称

添加账户

cd /data/svndata/winne/conf

编辑 authz 文件, 用户权限

[groups]
admin = bill,ada   
[/]
@admin = rw
[svndata:/document]
@admin     = rw

编辑 passwd 文件, 用户的密码

[users]
ada = ada
bill = 123123123123

编辑svnserve.conf , 确保有以下配置

[general]
auno-access = none
auth-access = write
password-db = passwd
authz-db = authz

重启生效

先 kill, 然后

svnserve -d -r /data/svndata/

尽量重命名, 不要 rm -rf

svn hooks

1 创建一个项目

svnadmin create /data/svndata/testhook

2 创建项目钩子脚本

$ cp /data/svndata/testhook/hooks/post-commit.tmpl /data/svndata/testhook/hooks/post-commit
$ chmod u+x /data/svndata/testhook/hooks/post-commit

3 为testhook仓库添加一个只读用户,用于更新目标仓库:

$ cd /data/svndata/testhook/conf

$ grep -Ev "(^$|^#)" authz
[groups]
hook = svnhook


[/]
@hook = r

[svndata:/test]
@hook = r


$ grep -Ev "(^$|^#)" passwd
[users]
svnhook = svnhook


$ grep -Ev "(^$|^#)" svnserve.conf
[general]
anon-access = none
auth-access = write
password-db = passwd
authz-db = authz
[sasl]

4 重启 svn, 使配置生效

5 到 nginx 目录下先 checkout 项目

$ cd /opt/www/svnhook/
$ svn co  svn://localhost/testhook

6 本地 checkout 仓库, 修改文件上传, 看是否触发钩子

遇到的坑

测试用的仓库是没问题了, 但是在文档仓库做这个配置之后, 发现不能实现更新, 一开始以为没触发钩子, 首先以为是权限问题, 把 post-commit 脚本改成了777, 把更新目录的用户也折腾了一下, 还是不行.

后来以为钩子没触发, 但是在钩子脚本中加入的测试日志表明, 是触发了钩子的

于是就去看错误日志, 发现有这样的报错:

cat /tmp/svnhook_error.log
Updating 'opt/www/svnhook/document':
svn: E000022: Error converting entry in directory '/opt/www/svnhook/testhook/php' to UTF-8
svn: E000022: Can't convert string from native encoding to 'UTF-8':
svn: E000022: ?\230?\149?\176?\230?\141?\174?\230?\168?\161?\229?\158?\139

原来是字符集的问题, locale看了一下系统的, 是 en_US, 但是文件中有中文, 所以就执行不了 update 了

解决方法 → 在脚本中加入:

export LC_ALL=zh_CN.UTF-8
